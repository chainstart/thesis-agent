# Iteration 003: 视觉大模型审阅接入

日期：2026-05-06

## 结论

当前修复稿不能判定为最终合格。第二轮只解决了空白页、页码重置、长空段和部分题注 keep 规则；经过渲染页面视觉审阅后，仍存在模板级差异和内容质量风险。

## 新增能力

- 新增 `vision-pack` 命令，生成视觉大模型审阅包：
  - 模板页面联系表。
  - 目标论文页面联系表。
  - 重点页面副本。
  - `vision_review_prompt.md` 审阅提示词。
- 审阅提示词要求视觉模型输出 `pass/fail`、按页列证据、按章节列内容问题和下一轮自动修复动作。
- 配置文件新增前置声明页为硬性结构检查：
  - 毕业设计（论文）学术诚信声明。
  - 毕业设计（论文）AI 使用情况声明。
  - 毕业设计（论文）版权使用授权书。

## 私有样本修复稿视觉审阅发现

审阅包：`runs/fixes/private-sample-vision-pack/`

### 格式问题

- FAIL：目标论文直接从摘要开始，缺少模板前 6 页所示的学术诚信声明、AI 使用情况声明、版权使用授权书等前置材料。
- FAIL：模板前置页的页眉横线、罗马页码、声明标题布局没有迁移到目标文档。
- FAIL：目录页标题被拆成上一页末尾的“目”和下一页的“录”，目录视觉效果不符合模板常规排版。
- FAIL：题注仍有跨页风险。第 20、23、36、37、39 页需要继续检查图表与题注绑定。
- FAIL：参考文献页排版松散，URL 显示为蓝色超链接，部分条目换行缩进不一致，第 45 页剩余大面积空白。

### 内容问题

- 测试章节仍偏薄，主要是功能现象描述，缺少测试环境、测试用例表、测试步骤、预期结果、实测结果、异常处理和结论分析。
- 需求分析章节与系统设计章节边界不够清晰，目录中 3.1 仍写成“系统技术需求分析”，与第 2 章标题重复。
- 参考文献中网页类来源比例偏高，且部分编号和来源质量需要人工复核。

## 下一步

1. 实现前置页移植或生成：从模板复制声明页结构，填充论文题目、作者、日期等占位内容。
2. 修复目录：生成完整自动目录，避免“目/录”拆页，保证页码与渲染结果一致。
3. 修复题注绑定：识别图、表对象，按照表题在表前、图题在图后的规则设置同页约束。
4. 规范参考文献：去除超链接蓝色样式，统一悬挂缩进和行距。
5. 生成内容增强建议，重点补充测试章节和需求分析章节。
